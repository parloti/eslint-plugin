# Documentation

This repository keeps package documentation in three areas:

- `docs/design/` for architecture, release-process, and maintenance notes.
- `docs/rules/` for rule-by-rule reference pages.
- `docs/api/` for autogenerated API reference from the public TypeScript entrypoint.

## Regenerating API Docs

Run `npm run docs` from the repository root. The command rebuilds `docs/api/` from `src/index.ts` using TypeDoc.

## Release Readiness

Before cutting or automating a release:

- confirm the design docs still reflect the current package structure
- confirm rule docs still match rule names and behavior
- regenerate `docs/api/`
- run `npm run validate` twice
