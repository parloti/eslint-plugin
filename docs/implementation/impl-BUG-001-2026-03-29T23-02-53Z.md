## 🧪 Notes

- Edge cases handled:
  - Different declaration shapes still get isolated typed fixture projects.
  - Cleanup still removes all temporary directories at the end of each suite.
- Reproduction/regression checks:
  - Focused prefer-vitest suites ran in about 10.5 seconds without coverage after the change.
  - The slow e2e prefer-vitest-incremental-casts suite dropped from roughly 30 to 32 seconds under full load to about 18 seconds.
- Assumptions:
  - Reusing one typed project per suite is acceptable because these tests are sequential and rewrite the same fixture file per case.
- Limitations:
  - Type-aware ESLint startup is still the dominant cost; this change removes repeated project boot rather than changing parser strategy.
