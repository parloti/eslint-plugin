## ⚠️ Issues

- [ ] The new `prefer-vi-mocked-import` rule only ever reports the first eligible mock in a file, so additional supported `vi.mock` or `vi.doMock` factories are silently skipped. The rule reports once from the `Program` listener in [src/domain/testing/prefer-vi-mocked-import/rule.ts](../../src/domain/testing/prefer-vi-mocked-import/rule.ts#L8), and its matcher explicitly collects only the first applicable top-level mock via `findMockStatement()` in [src/domain/testing/prefer-vi-mocked-import/match.ts](../../src/domain/testing/prefer-vi-mocked-import/match.ts#L102) and [src/domain/testing/prefer-vi-mocked-import/match.ts](../../src/domain/testing/prefer-vi-mocked-import/match.ts#L175). That means a file with two matching factories will pass with only one rewrite/report, which is a correctness gap rather than a style preference. Severity: High.

- [ ] Repository guidance is still partially stale after the layer refactor, so the repo is now teaching contributors to use a file that no longer exists. The current instructions still point contributors at `src/application/rules.ts` in [.github/copilot-instructions.md](../../.github/copilot-instructions.md#L12), but the live aggregation barrel is `src/application/index.ts`. This is the same class of drift called out in the previous review, and it will keep steering future changes toward a dead path. Severity: Medium.

- [ ] The repository boundary overlay still relies on the deprecated `elementTypes` configuration entry in [eslint.config.ts](../../eslint.config.ts#L21). Even though the repo now enforces the intended layer graph, this keeps the lint surface on a migration path and preserves avoidable warning noise in validation. Severity: Low.

---

## 💡 Suggestions

- Rework `prefer-vi-mocked-import` to analyze and report all supported matches in a file, or document and enforce a one-mock-per-file invariant if that is truly intended.
- Update the repo-local Copilot guidance so it references `src/application/index.ts` instead of the removed `src/application/rules.ts` path.
- Migrate the boundaries overlay off the deprecated `elementTypes` entry before the next dependency upgrade turns the warning into breakage.
